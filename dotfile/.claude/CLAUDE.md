# 全局工作偏好

## 语言

- 交流使用**中文**
- commit message 使用**中文**
- 代码变量名、函数名使用英文

## 沟通风格

- 用户习惯使用数字列表表达多步骤任务（如 `1, xxx 2, xxx`）
- 用户常给出简短确认（如 `确认`、`继续`、`直接实现`）
- 用户习惯迭代式工作：先给方向，再逐步调整细节, **大胆想象，小心求证**

## 任务执行

### 计划与实现

- 对于复杂任务，先理解上下文再动手
- 如需修改代码，先读取相关文件理解现有逻辑
- 遇到多种方案时，主动列出选项让用户选择

### 代码风格

- 保持简洁，避免过度工程化
- 遵循项目现有代码风格和约定
- 只在必要时添加注释

### Git 操作

- **不要自动 push**，除非用户明确要求
- 不要使用 `--no-verify` 跳过检查

## 工具使用

### LSP 工具

- `line` 和 `character` 参数都是 **1-based**（从1开始）
- 使用 LSP 前**必须先用 Read 工具确认位置**

### 搜索和探索

- 简单查询直接用 Grep/Glob
- 复杂探索任务使用 Explore agent

## 禁止事项

- 不要在 commit message 中添加 emoji
- 不要未经确认执行破坏性操作（删除文件、force push 等）
- 不要过度添加注释或文档，除非用户明确要求
